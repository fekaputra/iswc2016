Introduction

This year we have added an additional option to allow for the submission of articles formatted using HTML. Authors choosing to submit using HTML still need to comply with the page limit and layout requirements of the conference. It’s the author’s responsibility to ensure that reviewers can easily access their submission. This page provides a guide to helpful tools that can make HTML submissions easier.

Your HTML submission with any extensions if accepted will be available on the ISWC web page in preprint form with all the new elements that you’ve included.

General Guidance

HTML submissions should be submitted to EasyChair as a ZIP archive that contains the complete content of the paper. It should include a main “index.html” as well as any additional, referenced files..

This is an experiment! So help us help you. Please reach out to us if you have questions or need clarifications. We also ask you to do this with plenty of time before submission.

Pointers

The following tools can help produce well formatted academic papers using HTML; they are capable of rendering papers in the LNCS layout, and a conversion to LaTeX is possible:

  • Research Articles in Simplified HTML (RASH) format to simplify writing of documents (example paper)
  • dokieli – decentralized authoring, annotations and social interaction tool for Linked Research HTML documents (example paper)
Other pointers are hosted by the W3C Scholarly HTML community group.
